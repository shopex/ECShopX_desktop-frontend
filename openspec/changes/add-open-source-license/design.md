## Context
项目即将开源，需要在所有源代码文件中添加开源许可证备注。这是一个跨文件的大规模修改，需要自动化处理以确保一致性和准确性。

## Goals / Non-Goals
- Goals: 
  - 为所有源文件添加统一的开源许可证备注
  - 确保备注格式专业且符合开源标准
  - 自动化处理避免手动错误
- Non-Goals:
  - 不修改现有代码逻辑
  - 不改变文件结构
  - 不处理非源代码文件

## Decisions
- Decision: 使用MIT许可证作为开源许可证
- Alternatives considered: Apache 2.0, GPL v3
- Rationale: MIT许可证简单明了，兼容性好，适合商业项目

- Decision: 备注格式使用标准的多行注释
- Alternatives considered: 单行注释，文档注释
- Rationale: 多行注释清晰易读，支持版权信息

## Risks / Trade-offs
- 风险: 大量文件修改可能导致版本控制冲突
- 缓解: 使用自动化脚本，确保修改一致性

- 风险: 某些文件可能已有备注，需要智能处理
- 缓解: 脚本检测现有备注，避免重复添加

## Migration Plan
1. 创建自动化脚本
2. 在测试分支验证脚本效果
3. 批量处理所有源文件
4. 验证项目功能正常
5. 提交到主分支
